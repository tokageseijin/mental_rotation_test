# デザイン根拠（Design Rationale）

「なぜこのUIなのか」を、厳しいデザインレビューでも説明できるよう明文化する。
実装トークンは [`src/styles/tokens.css`](../src/styles/tokens.css) と一致させている。

## 1. 目的とデザイン原則

このアプリの本質は **立体を頭の中で回す（メンタルローテーション）ことに集中させる** こと。
UI自体が主役ではない。したがって全体を貫く原則は次の3つ。

1. **Content-first / 低ノイズ**: 画面上で最も目立つべきは3Dオブジェクトと選択肢。
   装飾・色・境界線は最小限にして、視線を課題へ誘導する。
2. **Recognition over recall（想起より認識）**: 機能は常に同じ場所に置き、探させない。
3. **Progressive disclosure（段階的開示）**: 習熟度や詳細分析は必要なときだけ深掘りできる。

## 2. 色 — 「白系シンプル」の具体化

- 面は白/オフホワイトの3階層（`--surface-app` / `--surface-card` / `--surface-sunken`）。
  **色ではなく明度差と余白**で階層を表現するため、彩度ノイズが出ない。
- **アクセントは青1色**（`--accent`）。1画面の主要アクション（例:「クイズを開始」）を
  1つに絞る Hick's law 的な意思決定負荷の低減。
- 成否のみ意味色（緑/赤）を使用。色覚多様性に配慮し、色だけに依存せず
  **枠線＋テキスト（「正解」）** を併記する（アクセシビリティ）。
- 本文・主要操作のコントラストは **WCAG AA（4.5:1）以上** を白背景基準で満たす。

## 3. レイアウトとナビゲーション

- **左サイドの固定ナビ**（ライブラリ / クイズ / 成績 / 設定）。
  - 根拠: 利用者は少数の目的地を反復して行き来する。固定の空間メニューは
    ハンバーガー等の隠れたナビより発見性が高く、常時1クリックで到達できる。
  - 狭い画面ではナビを上部バーに畳み、親指到達性を確保（レスポンシブ）。
- コンテンツは最大幅を設けて長い行長を避け、可読性（行長50–75字目安）を保つ。

## 4. 間隔・タイポグラフィ

- **8pxスペーシンググリッド**（4px半ステップ）でリズムを統一。恣意的な余白を排除。
- タイプスケールは 12/14/16/20/28 の5段階に限定。見出し・本文・補助の役割を明確化。

## 5. 主要アクションと Fitts の法則

- 「クイズを開始」「次の問題」など主要CTAは **大きめのヒット領域**（`.btn.lg`）にし、
  幅いっぱい（`.btn.block`）で押しやすくする。ペン/タッチでも誤タップしにくい。

## 6. ポインタ／ペン対応の設計

- 全操作を **Pointer Events** に統一（mouse/pen/touch を単一経路で処理）。
- ページ全体は `touch-action: manipulation` を敷き、**300ms のタップ遅延と
  ダブルタップズームを抑制** しつつ、ペンのタップを通常クリックとして機能させる。
- 3Dビューア／描画キャンバスなどジェスチャを占有する領域だけ `touch-action: none`。
- ペンの筆圧（`pressure`）は描画モード（フェーズB）で線幅に反映。

## 7. フィードバックと学習効果

- 回答後は **正解の緑・誤答の赤** を即時に提示し、さらに *どの種類の誤り* かを言語化
  （軸違い/符号違い/量違い/グローバル・ローカル取り違え/鏡像）。
  - 根拠: 単なる正誤より、誤りの帰属（attribution）が学習転移を高める。
- 「易しいのに間違えた／難しくて間違えた」を区別表示し、次に何を練習すべきかを示す。

## 8. データの所有権と透明性

- 設定・成績・モデル一覧はすべてブラウザ内に保存し、**エクスポート/インポート** を提供。
  ユーザーが自分のデータを完全にコントロールできる（ロックインしない）。
- ブラウザはOSのファイルパスを保持できないため、ユーザーモデルは File System Access の
  ハンドル（対応ブラウザ）または内容バイト列として保存し、その制約を設定画面に明示する。

## 9. 今後の検討（フェーズB）

- ドローイングモードのキャンバスUI（筆圧・パームリジェクション）と自己評価フロー。
- 誤答分析のチャート化、キーボード操作の網羅（A–Dキーで選択肢を選ぶ等）。
